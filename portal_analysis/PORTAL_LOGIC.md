# Логика заполняемости полей портала invest.gov.ru

> Источник истины для `portal_checker.py`.
> Основан на «Методических рекомендациях по внесению данных на инвестиционную карту РФ» (29.03.2024).
> При любом изменении логики — сначала обновить этот файл, потом код.

---

## 1. Типы полей

| Тип | Описание | Пример |
|---|---|---|
| `text` | Свободный текст | Название площадки |
| `number` | Число | Свободная площадь ЗУ, га |
| `dropdown` | Выпадающий список, не порождает дочерних полей | Формат площадки, Тип площадки |
| `dropdown_conditional` | Выпадающий список, от значения зависят дочерние поля | Преференциальный режим, Наличие ж/д |
| `child` | Поле, появляющееся только при определённом значении родителя | Тариф на потребление воды |
| `file` | Загрузка файла | Фотографии объекта |
| `geo` | Геопривязка | Геопривязка |

## 3. Что портал считает «заполненным»

| Тип поля | Считается заполненным | Не считается заполненным |
|---|---|---|
| `text` | Любой непустой текст (после strip HTML) | Пусто, пробелы, «не указано», «не заполнено» |
| `number` | Любое число, включая 0 | Пусто, null |
| `dropdown` | Любое значение из списка, включая «нет» | Пусто |
| `dropdown_conditional` | Любое значение из списка | Пусто |
| `child` | Зависит от типа дочернего поля | — |
| `file` | Файл загружен | Файл отсутствует |
| `geo` | Координаты указаны | Не указаны |

> ⚠️ Открытый вопрос: считается ли «Не применимо» в текстовых полях (напр. «Описание процедуры») заполненным?
> Пока «не применимо» входит в IRRELEVANT_VALUES — уточнить по площадке 1522201.
