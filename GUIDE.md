# GUIDE.md — Руководство пользователя KITEZH

> Этот документ — пошаговая инструкция по запуску, обновлению и резервному копированию.  
> Описание архитектуры и структуры кода — в [README.md](README.md).

---

## Первый запуск

### Шаг 1 — Установка

1. Распаковать архив портативной сборки (`KITEZH_vX.X.zip`) в рабочую папку.
2. Запустить **`install.bat`** — скрипт установит все Python-зависимости в локальный `WPy/`.
3. Создать файл `.env` в корне папки:

```env
SECRET_KEY=любая_случайная_строка_длиной_32+_символа
GITHUB_TOKEN=ghp_...    # нужен только для автообновления
```

> Если нет `GITHUB_TOKEN` — автообновление недоступно, всё остальное работает.

### Шаг 2 — Запуск

1. Дважды кликнуть **`start KITEZH.bat`** (или запустить из командной строки).
2. В системном трее появится иконка KITEZH.
3. Открыть браузер: [http://localhost:5000](http://localhost:5000)
4. Войти под учётной записью.

> Если браузер не открывается автоматически — введи адрес вручную.

### OCR (опционально)

Для работы с OCR-распознаванием запусти `install_ocr.bat`.  
Требуется ~500 МБ свободного места (модели PyTorch).  
Система работает без OCR — эта функция опциональна.

## Обновление системы

Обновление выполняется через веб-интерфейс без остановки работы.

### Через веб-интерфейс (рекомендуется)

1. Перейти в **Администрирование → Обновление системы** (`/admin/update`).
2. Нажать **«Проверить обновления»** — система покажет доступную версию.
3. Нажать **«Обновить»** — начнётся процесс:
   - Сервер перейдёт в режим обслуживания (`/maintenance`).
   - Пользователи увидят заглушку с таймером обратного отсчёта.
   - Патчи скачиваются и применяются в фоне.
   - После завершения сервер перезапускается автоматически.
4. Обновление занимает 1–3 минуты.

> ⚠️ База данных (`database.db`), загруженные файлы (`uploads/`) и отчёты (`reports/`) **не затрагиваются** при обновлении.

### Через командную строку (ручное)

```
update.bat
```

Требует `GITHUB_TOKEN` в `.env`. Подходит при проблемах с веб-интерфейсом.

## Резервное копирование

### Автоматический бэкап

Планировщик (`services/backup_scheduler.py`) создаёт резервные копии автоматически.  
Копии сохраняются в папку `backups/` в формате `backup_YYYY-MM-DD_HH-MM.zip`.

### Ручной бэкап

Скопировать вручную из папки проекта:
- `database.db` — база данных
- `uploads/` — загруженные документы
- `reports/` — сгенерированные отчёты
- `.env` — настройки окружения

### Восстановление из резервной копии

1. Разместить резервную копию рядом с папкой проекта как `KITEZH.backup/`.
2. Запустить **`copy_data.bat`**.
3. Скрипт скопирует `.env`, `database.db`, `uploads/` и `reports/` в рабочую папку.

> Перед восстановлением — остановить сервер: закрыть иконку KITEZH в трее или завершить процесс `start KITEZH.bat`.

## Новая структура проекта

| Папка / файл | Назначение |
|---|---|
| `app.py` | Точка входа, запуск Flask, регистрация Blueprints |
| `core/` | Инфраструктура: авторизация, журнал, логи, rate-limiting |
| `routes/` | HTTP-маршруты (Flask Blueprints) — по одному файлу на модуль |
| `services/` | Бизнес-логика: дашборд, OCR, экспорт, бэкапы |
| `utils/` | Переиспользуемые утилиты: валидация, формы, GitHub API |
| `updater/` | Подсистема автообновления: ядро, changelog, шаблон БД |
| `requests_app/` | Пакет обращений (list / form / view / action) |
| `templates/` | HTML-шаблоны Jinja2 |
| `static/` | CSS, JS, изображения |

> Полное дерево файлов с описанием каждого модуля — в [README.md](README.md).

## Часто задаваемые вопросы

**1. Сервер не запускается после `start KITEZH.bat`**  
Проверь наличие файла `.env` в корне проекта. Убедись, что `SECRET_KEY` задан.  
Посмотри логи в папке `logs/` — там будет причина ошибки.

**2. Страница зависла на «Режим обслуживания»**  
Подожди 3–5 минут — обновление может занять время при медленном интернете.  
Если зависло дольше 10 минут — перезапусти `start KITEZH.bat` вручную.

**3. После обновления пропали данные**  
Данные не удаляются при обновлении. Проверь, что `database.db` находится в папке `db/`.  
Если нужно восстановить — используй `copy_data.bat` из резервной копии.

**4. OCR не работает / не распознаёт документы**  
Убедись, что запущен `install_ocr.bat` и установлены зависимости.  
OCR требует ~500 МБ свободного места. Системный Python не подойдёт — нужен WPy.

**5. Как добавить нового пользователя?**  
Перейти в **Администрирование → Пользователи** (`/admin/users`).  
Нажать «Добавить» и заполнить форму. Права доступа задаются ролью.
